의식의 흐름대로 글쓰기를 즐기며, 읽기 쉬운 글을 못 쓰지만 읽기 쉬운 코드 작성을 위해 노력하는 웹 퍼블리셔입니다.

Semantic markup, 올바른 웹 접근성 구조 설계, CSS/JS 구조와 표현의 분리 등에 관심이 많아 꾸준히 학습을 하고 있고 많이 알지는 못하지만 실무에 적용하여 보다 더 나은 웹을 만들고자 노력합니다.

주요 관심사에 대한 포스트

  • 순수 Javascript로 레이어팝업 초점이동을 구성해보자
  • 모바일 웹 접근성) abbr 요소
  • sub-pixel rendering 이슈에 대응하는 최선의 방법은 무엇인가
  • 레이어팝업에 접근성을 불어넣다
  • 접근성이 충분히 보장된 탭 UI, 만들어보자

직접 만든 블로그

학습한 것을 바탕으로 제 블로그를 직접 설계, 퍼블리싱하였습니다(관련 포스트). 처음에는 대부분의 블로거처럼 타인이 만든 블로그 템플릿을 가져와 쓰는 것부터 시작했는데, 아무래도 입맛에 맞게 커스터마이징하기에는 한계가 있다고 느껴서 직접 만들었습니다. 직접 만들고 유지보수하면서 학습되는 게 참 많습니다. 그래서 더 의미있는 포스팅을 하고 있다고 생각합니다.

Jekyll 공식 템플릿 목록에 아직 올리지 않았으나 더 개선한다음 남들이 사용할 수 있게 추후 올릴 예정입니다. 템플릿 github 주소이며, 피드백 환영합니다.
